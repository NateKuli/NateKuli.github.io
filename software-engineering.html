Enhancement Narrative

My CS-340 Project Two dashboard, which was initially made for Grazioso Salvare, a business that specialized in training and placing rescue dogs, was the artifact I chose for improvement. The dashboard's objective was to offer an interactive method of examining animal outcome data from the Austin Animal Center. In the initial edition, visitors could look at breed distribution and geographic area, access comprehensive records, and filter dogs by rescue type. In order to provide a polished and user-friendly experience, it featured dynamic data displays, interactive widgets, and visual charts that were all branded with my name and the Grazioso Salvare logo. I selected this artifact because it showed a number of critical abilities, such as database interaction, data visualization, and user interface creation, and it offered a solid basis for implementing the software engineering best practices needed for the CS-499 capstone.

My improvements were concentrated on enhancing general code quality, security, stability, and maintainability. Using environment variables kept in a .env file in place of hard-coded MongoDB credentials was one of the biggest gains. This modification brought the project into compliance with secure coding standards and made sure that private information was no longer visible in the source code. Additionally, I added structured logs for database transactions, callback execution, and data loading, and I built Python's logging module to replace print statements. I added error handling throughout the dashboard to increase reliability. This includes try/except blocks that avoid crashes and offer user-friendly fallback behavior when data is wrong or missing. In order to make the code easier to comprehend and maintain, I also rearranged the notebook into logical sections and enhanced the documentation by adding a professional header, docstrings, and clearer comments.

To increase callback dependability, a number of improvements were performed. In order to avoid frequent runtime mistakes and guarantee that the dashboard operated consistently even when the dataset changed, I implemented validation tests for empty DataFrames, missing columns, and invalid row choices. In addition, I eliminated unnecessary logic, tidied up variable names, and made sure the notebook's formatting was consistent throughout. Together, these enhancements made the dashboard more reliable, safe, and in line with actual software engineering requirements.

I ran across a number of difficulties during the enhancement procedure. I completely rebuilt the dashboard in Visual Studio Code using the accessible code as I was unable to access the original Codio environment. This necessitated meticulously rebuilding the notebook structure and personally testing every part. By changing the password and shifting the configuration into environment variables, I was also able to fix authentication problems brought on by incorrect MongoDB credentials. Map index issues were repaired by verifying the selected row and making sure the necessary fields were present before rendering the map, and a crash caused by trying to remove the _id column from an empty DataFrame was fixed by adding a conditional check. The dashboard also failed because of a missing image file, which I fixed by changing the filename and making sure the image was in the project directory. These difficulties improved my debugging abilities and reaffirmed the need of defensive programming.

Opening the dashboard will launch the improved dashboard.ipynb file in Visual Studio Code or any Jupyter Notebook environment, executing each cell one after the other and filtering the data with the radio buttons. The pie chart and the map are both dynamically updated when a row is selected. The improved version maintains all intended functionality and offers a dependable, interactive interface for examining rescue dog data, notwithstanding the unavailability of the original Codio environment. My progress in secure code, error management, documentation, and general software engineering techniques is evident in the enhancements I made. My technical skills and preparedness for professional software development are now well-represented by this improved artifact.
