Enhancement Narrative

I chose my DAD 220 Project One SQL item, which was first produced in early 2024, to improve my databases. The basic database schema, SQL queries, and data manipulation techniques needed to operate a small business inventory and customer-order system are contained in this item. Basic SQL skills, such as table creation, primary and foreign key relationships, and basic SELECT, INSERT, UPDATE, and DELETE operations, were demonstrated in the original project. I chose this artifact because it reflects my initial work with relational databases, and improving it offers a great chance to show how my abilities in database design, normalization, query optimization, and secure data handling have developed.

I added this artifact to my ePortfolio because it closely relates to CS-499 Outcome 4, which emphasizes applying creative and well-founded methods, abilities, and resources in computing practices to produce valuable solutions. Almost all contemporary software systems rely on databases, and this item demonstrates my abilities to create structured data models, maintain relational integrity, and create effective SQL queries. In order to improve the artifact, I updated a number of queries to increase readability and efficiency, implemented constraints to ensure data integrity, and normalized a number of tables to decrease redundancy. To illustrate safe procedures that aid in preventing SQL injection vulnerabilities, I included instances of parameterized queries. These improvements transform the artifact from a simple academic exercise into a professional database solution that is in line with industry standards.

The improvements I made are in line with the course objectives I intended to cover in Module One. My objectives were to showcase better database design, implement normalization principles, bolster security considerations, and increase the performance and clarity of SQL queries. With the improvements mentioned above, I was able to accomplish these objectives. I do not need to alter my outcome-coverage strategy because the accomplished work is in line with the databases category's desired results.

As I thought back on the improvement process, I realized how crucial careful database architecture is to system efficiency and long-term maintainability. I was able to identify areas where the schema could be enhanced, such as removing needless duplication and enforcing stricter relational constraints, by going back and reviewing my initial item. Finding a way to reorganize the schema without disrupting the logical connections between tables was one of the difficulties I faced. Rewriting queries to be both effective and understandable presented another difficulty, particularly when adding more sophisticated SQL features. My comprehension of normalization, indexing, safe query techniques, and the function of databases in sustaining dependable software systems has improved as a result of this approach. Improving this artifact allowed me to advance as a developer and gain a deeper understanding of the significance of clear, organized data models.

Overall, this enhancement demonstrates meaningful progress toward competency in database design, secure data handling, and professional-quality implementation. It represents a significant improvement over the original DAD 220 project and serves as a strong example of my ability to work with relational databases in a professional context.
